## 🌱 Vanilla JavaScript
- **DOM (Document Object Model)**  
  - *What:* The tree-like structure of a webpage (HTML elements).  
  - *Why:* Vanilla JS manipulates the DOM directly (`document.querySelector`, `innerHTML`).  
  - *When:* Small scripts, learning fundamentals.

- **Event Listeners**  
  - *What:* Functions that respond to user actions (clicks, typing).  
  - *Why:* Core of interactivity.  
  - *When:* Any interactive webpage.

---

## ⚛️ React
- **Declarative Rendering**  
  - *What:* You describe *what* the UI should look like, React figures out *how* to update the DOM.  
  - *Why:* Easier to reason about, less manual work.  
  - *When:* Complex UIs with changing state.

- **Reusable Components**  
  - *What:* Small UI blocks (buttons, forms) reused across app.  
  - *Why:* Consistency, maintainability.  
  - *When:* Apps with repeated UI patterns.

- **Virtual DOM**  
  - *What:* A lightweight copy of the real DOM React uses to calculate changes.  
  - *Why:* Faster updates, avoids costly DOM operations.  
  - *When:* Any React app.

---

## 🚀 Next.js
- **File-Based Routing**  
  - *What:* Each file in `/pages` becomes a route automatically.  
  - *Why:* No manual route setup.  
  - *When:* Multi-page apps.

- **SSR (Server-Side Rendering)**  
  - *What:* Pages rendered on server before reaching browser.  
  - *Why:* Faster initial load, better SEO.  
  - *When:* Blogs, e-commerce, SaaS.

- **SEO (Search Engine Optimization)**  
  - *What:* Making pages discoverable by search engines.  
  - *Why:* Server-rendered HTML is easier for Google to index.  
  - *When:* Public-facing sites.

- **API Routes**  
  - *What:* Backend endpoints inside Next.js (`pages/api/...`).  
  - *Why:* Handle server logic without separate backend.  
  - *When:* Small apps needing auth, forms, or simple APIs.

- **React Server Components (RSC)**  
  - *What:* Components that run on server, not browser.  
  - *Why:* Less JavaScript shipped → faster loads.  
  - *When:* Performance-focused apps.

---

## ⚙️ Express.js
- **Middleware**  
  - *What:* Functions that run between request and response (logging, auth).  
  - *Why:* Flexible way to add features.  
  - *When:* Any backend API.

- **Routing**  
  - *What:* Defining endpoints (`app.get('/users', ...)`).  
  - *Why:* Organizes API logic.  
  - *When:* REST APIs, small servers.

---

## 🛡️ Nest.js
- **Opinionated**  
  - *What:* Enforces a specific structure (modules, controllers, services).  
  - *Why:* Consistency across large teams.  
  - *When:* Enterprise apps.

- **TypeScript-First**  
  - *What:* Built with TypeScript in mind.  
  - *Why:* Type safety, fewer bugs.  
  - *When:* Large projects.

- **Dependency Injection**  
  - *What:* System automatically provides required services to classes.  
  - *Why:* Cleaner, testable code.  
  - *When:* Complex apps.

- **Modular Architecture**  
  - *What:* Code split into modules (auth, users, payments).  
  - *Why:* Easier to maintain and scale.  
  - *When:* Enterprise backends.

---

## 🔗 Socket.io
- **WebSockets**  
  - *What:* Protocol for continuous two-way communication.  
  - *Why:* Unlike HTTP, stays open for live data exchange.  
  - *When:* Chat, live updates, multiplayer games.

- **Real-Time Communication**  
  - *What:* Data updates instantly without refresh.  
  - *Why:* Enables chat, notifications, collaborative editing.  
  - *When:* Any app needing instant updates.

---

## 💻 Electron.js
- **Cross-Platform**  
  - *What:* Runs on Windows, Mac, Linux.  
  - *Why:* One codebase for all desktops.  
  - *When:* Desktop apps like VS Code, Slack.

- **Reuse Web Skills**  
  - *What:* Build desktop apps with HTML, CSS, JS.  
  - *Why:* No need to learn native desktop languages.  
  - *When:* Teams with web developers.

---

## 📱 React Native
- **Shared Codebase**  
  - *What:* Write React code, reuse across iOS and Android.  
  - *Why:* Faster development, less duplication.  
  - *When:* Mobile apps.

- **Native-Like Performance**  
  - *What:* Bridges React components to native mobile UI.  
  - *Why:* Feels like a real native app.  
  - *When:* Apps needing smooth mobile experience.

---

## 🎨 Three.js / D3.js
- **WebGL (Three.js)**  
  - *What:* Low-level graphics API in browsers.  
  - *Why:* Enables 3D rendering.  
  - *When:* Games, 3D models.

- **Data Binding (D3.js)**  
  - *What:* Connects data to DOM elements.  
  - *Why:* Makes dynamic charts possible.  
  - *When:* Dashboards, analytics.

---

## 🗺️ Summary
- **Vanilla JS:** DOM, events.  
- **React:** Declarative rendering, Virtual DOM, reusable components.  
- **Next.js:** SSR, SEO, API routes, RSC, file-based routing.  
- **Express:** Middleware, routing.  
- **Nest.js:** Opinionated, TypeScript-first, modular, dependency injection.  
- **Socket.io:** WebSockets, real-time communication.  
- **Electron:** Cross-platform, reuse web skills.  
- **React Native:** Shared codebase, native-like performance.  
- **Three.js/D3.js:** WebGL, data binding.
